O tema escolhido para o projeto foi o tema 4, o mesmo com as seguintes exigencias e/ou sugestões:
Tema 4 – Receitas Culinárias Saudáveis

Estrutura de dados:
Lista duplamente ligada de Receitas
Cada receita possui uma lista duplamente ligada de Ingredientes Saudáveis

Funções básicas:
Cadastrar receita (Feito)
Remover receita (Feito)
Listar receitas (Feito)
Cadastrar ingrediente em uma receita (Feito)
Listar ingredientes de uma receita (Feito)

Funções extras (exemplos):
Contar ingredientes por receita (Feito)
Listar receitas por tipo (vegana, vegetariana, sem açúcar, etc.) (Não Feito)
Calcular valor calórico total da receita (Não Feito)

Cruzamento de dados (exemplos):
Identificar ingredientes utilizados em mais de uma receita (Feito)
Listar receitas que utilizam um determinado ingrediente (Feito)
Encontrar a receita com menor valor calórico (Feito, mas com o menor preço, pois o valor calórico não foi implementado)
Calcular o ingrediente mais utilizado entre todas as receitas (Feito)


Toda a lógica do projeto gira em torno da manipulação de duas listas:
- A Lista No em lista.h , que é uma lista duplamente ligada, que salva nome, preço e ingredientes de cada receita.
- A Lista Receita em ingrediente.h , que é uma lista encadeada simples , que salva os ingredientes de forma linear.
Apartir da criaçãoo dessas duas listas, foi possivel implementar as funções exigidas, com controle sobre as informações
de cada receita.

Detalhamento das funções implementadas:

1. As funções do Menu interativo são bem simple, somente loops e switch case para controlar as opções do 
usuário, chamando as funções correspondentes para cada ação. Ao mesmo tempo que exibirMenu() somente possui varios printf em
sequencia, para mostrar as opções disponíveis para o usuário.

2. CriaLista() é uma função que aloca um ponteiro para a estrutura do No principal, depois de alocar memória.

3. CadastrarReceita(Lista *l) é praticamente a função mais complexa e importante na implementação.
Inicialmente ela cria um novo nó e pede ao usuario para inserir:
- O nome da receita (recebe uma string e tira o \n)
- O preço da receita (scanf comum)
- O número de ingredientes que a receita possui (o numero vai controlar o loop que pede ingredientes)
O loop dos ingredientes recebe a cada iteração o nome do ingrediente
o nome é guardade em uma varivel buffer, que remove o \n também,
então é criado uma lista Receita para guardar os ingredientes
nessa Lista é aplicado o valor de buffer, e então há uma verificação para ver se a 
lista de ingredientes na lista inicial No está vazia
- se vazia, aplica o valor da lista de ingredientes criada
- se não vazia, percorre a lista de ingredientes inteira até o prox = NULL, e copia o valor da Lista.
então o No criado, após receber nome, preço e ingredientes, é adicionado no inicio da lista de receitas, 
substituindo o ponteiro do inicio da lista para o novo nó criado.

4. cadastraringredientereceita(Lista lista, char nomeprato) é responsável por adicionar um ingrediente novo em uma receita que já está salva.
inicialmente ela verifica se a lista não é nula e se tem pelo menos uma receita cadastrada. em seguida, faz uma busca sequencial na lista duplamente ligada comparando o nome da receita usando strcmp.
quando a receita correspondente é encontrada, pede ao usuário para inserir o novo ingrediente:
- o valor é lido com fgets e o \n é removido
- é criado um novo nó dinamicamente para a lista simples de ingredientes
- a string do ingrediente é alocada com strdup para segurança na memória
então há uma verificação para ver se a lista de ingredientes daquela receita está vazia:
- se vazia, o novo nó passa a ser o primeiro elemento da lista
- se não vazia, o novo ingrediente é inserido no final da lista, preservando a ordem que foi digitada.
a função encerra logo após inserir o ingrediente na receita.

5. listaringredientesreceita(Lista lista, char nomeprato) tem como objetivo listar todos os ingredientes de uma receita específica.
primeiramente ela verifica se a lista existe e se contém elementos. depois, faz uma busca sequencial na lista de receitas até achar o nome informado.
ao localizar a receita:
- inicia um ponteiro auxiliar para percorrer a lista simples de ingredientes
- percorre a lista imprimindo cada ingrediente na tela até o ponteiro atingir null
a função retorna logo após exibir tudo. caso não ache a receita, ela simplesmente finaliza sem imprimir mais nada.

6. menorpreco(Lista lista) percorre toda a lista de receitas para identificar qual possui o menor preço.
inicialmente ela verifica se a lista é válida e define o primeiro nó como a referência inicial de menor preço.
em seguida ela entra em um loop:
- percorre a lista de receitas inteira
- a cada iteração, compara o preço atual com o menor valor salvo
- caso o preço seja menor, atualiza o ponteiro da receita mais barata e a variável do menor preço
ao final da varredura, imprime o nome da receita e o valor formatado com duas casas decimais. a complexidade é o(n), pois percorre a lista inteira apenas uma vez.

7. estrutura de frequência e liberação (freq, registraringrediente e liberarfreq) são usadas para mapear quantas vezes cada ingrediente aparece no geral.
a estrutura freq contém:
- nome do ingrediente
- contagem de quantas vezes ele aparece
- ponteiro para o próximo elemento da lista
a função registraringrediente faz o seguinte:
- percorre a lista de frequência para ver se o ingrediente já existe
- se encontrado, incrementa o contador
- se não existir, cria um novo nó com contagem 1 e insere no início da lista para melhorar o desempenho.
já a função liberarfreq serve para limpar a memória. ela percorre a lista inteira dando free em cada nó, evitando vazamentos.

8. registraringrediente(freq** listafreq, char* nomeingrediente) é o núcleo do sistema de contagem de frequência.
ela recebe o ponteiro duplo da lista e o nome do ingrediente a ser contabilizado.
o funcionamento ocorre em etapas:
- percorre a lista de frequência existente
- compara os nomes usando strcmp
- se o ingrediente já estiver lá, incrementa a contagem e encerra
caso não encontre, ela aloca um nó novo, define a contagem como 1 e insere esse nó no início da lista.
o ponteiro duplo serve para conseguir modificar diretamente o início da lista. essa estrutura é o que permite fazer funções como ingredientemaisusado e ingredientesemcomum.

9. IngredienteMaisUsado(Lista* lista)
utiliza a lógica de mapeamento de frequência para determinar qual item aparece mais vezes no sistema. A função cria uma lista de frequência temporária e percorre a lista duplamente ligada de receitas, acessando cada sublista de ingredientes e enviando cada nome para a função de registro. Após processar todos os dados, ela faz uma varredura na lista de frequência para encontrar o maior contador acumulado, exibindo o nome e o total de repetições antes de liberar a memória.

10. IngredientesEmComum(Lista* lista)
identifica quais componentes são compartilhados por mais de uma receita. Ela percorre a estrutura de frequência junto com a lista de receitas e suas listas de ingredientes. No momento da exibição, a função filtra apenas os nós da lista de frequência que possuem um contador superior a 1 e lista para o usuário. Se não ouver ingredientes com o numero superior a 1, informa ao usuário que não foram encontrados ingredientes repetidos entre os pratos cadastrados.

11. liberarIngredientes(Receita* head)
Como cada receita aloca seus ingredientes de forma dinâmica, a função recebe o ponteiro inicial da sublista e utiliza um laço de repetição para percorrer todos os nós. Um ponteiro auxiliar armazena o endereço do próximo elemento antes de o nó atual ser liberado com free, garantindo que nenhum endereço de memória seja perdido durante o processo de limpeza e evitando o vazamento de memória.

12. removerReceita(Lista* lista, char* nome)
localiza o nó alvo na lista principal comparando os nomes com strcmp. Ao encontrar o registro, a função chama obrigatoriamente a liberarIngredientes para limpar a sublista vinculada. Em seguida, reajusta os ponteiros dos nós vizinhos para que o anterior aponte para o sucessor e o sucessor aponte para o anterior. Se o nó for o início da lista, o ponteiro de controle é atualizado para o próximo elemento. Por fim, a memória do nó da receita é liberada.

13. listarReceitas(Lista* lista)
Inicialmente analisa se a lista está vazia, caso esteja retorna uma mensagem avisando o usuário, caso contrário vai para a segunda etapa.
Depois simplesmente cria um No secundario, igual ao da lista do argumento que fica em um loop para percorrer todas as receitas, cada iteração do loop 
informa: nome da receita, preço e printa cada um dos ingredientes, que está numa lista que é percorrida até prox = NULL. terminado uma receita, troca o No para prox e reinicia o loop.


14. listarReceitasPorIngrediente(Lista* lista, char* ingrediente)
Esta função tambem verifica a lista vazia.
cria um No secundario para perccorrer toda a lista, mas também inicia um int para falar se o ingrediente foi encontrado ou não.
igual a função listarReceitas , tem dois loops, o primeiro percorre todas as receitas, e o segundo todoss os ingrediente de cada receita.
nesse segundo loop, caso algum dos ingredientes for igual ao solicitado, muda found para 1 depois de printar para o usuario qual receita tem tal ingrediente.
caso nao seja encontrado o ingrediente em nenhuma receita, a função if(!found) verifica se found é 0, se for avisa que não tem nenhuma receita com o ingrediente.
